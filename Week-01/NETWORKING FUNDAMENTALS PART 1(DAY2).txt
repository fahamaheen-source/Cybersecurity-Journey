NETWORKING INTRODUCTION
What is Networking?
Networking is the process of connecting two or more devices so they can communicate and share data, resources, and services.
Examples of Network Devices
•	Computer
•	Laptop
•	Mobile Phone
•	Server
•	Printer
•	Router
________________________________________
Why Networking Matters in Cybersecurity
Networking is the foundation of cybersecurity because almost every cyber attack occurs through a network.
A SOC Analyst uses networking knowledge to:
•	Monitor network traffic
•	Detect suspicious activities
•	Investigate security alerts
•	Identify attackers
•	Protect organizational systems
Without networking knowledge, a SOC Analyst cannot effectively investigate incidents.
________________________________________

Devices communicate by sending and receiving data packets over a network.
Each device has a unique IP Address, which helps identify it on the network.

OSI MODEL
Definition
OSI (Open Systems Interconnection) Model is a 7-layer reference model that explains how data is transmitted from one device to another over a network.
Purpose of the OSI Model
•	Standardizes network communication
•	Helps different devices communicate
•	Simplifies troubleshooting
•	Helps understand where network problems occur
•	Used for learning, troubleshooting, and cybersecurity
________________________________________
OSI MODEL LAYERS
Layer	Name	Purpose	Example	Device/Protocol
7	Application	Provides network services to applications	Web browsing, Email	HTTP, HTTPS, FTP, DNS
6	Presentation	Formats, encrypts, compresses data	HTTPS encryption	SSL/TLS, JPEG, PNG
5	Session	Establishes, manages, and terminates sessions	Video calls	NetBIOS, RPC
4	Transport	Reliable data delivery	File download	TCP, UDP
3	Network	Routes packets using IP addresses	Internet routing	IP, Router
2	Data Link	Transfers data within a LAN using MAC addresses	Local communication	Ethernet, Switch
1	Physical	Transmits raw bits over physical media	Network cables	Cable, Fiber, Wi-Fi, NIC

TCP/IP MODEL
Definition
TCP/IP (Transmission Control Protocol/Internet Protocol) is a 4-layer networking model used for communication over the Internet.
It is the practical networking model used by almost all modern computer networks.

IP ADDRESSING
Definition
An IP (Internet Protocol) Address is a unique numerical address assigned to every device connected to a network. It allows devices to identify and communicate with each other.
Example:
•	192.168.1.10
•	10.0.0.5
•	8.8.8.8
________________________________________
Why IP Addresses Matter
IP addresses are important because they:
•	Identify devices on a network.
•	Enable communication between devices.
•	Help routers deliver data to the correct destination.
•	Allow users to access websites and online services.
Without IP addresses, devices cannot communicate.
________________________________________
IPv4 Basics
IPv4 (Internet Protocol Version 4) is the most widely used IP addressing system.
Features
•	32-bit address
•	Four numbers separated by dots
•	Each number ranges from 0 to 255
Examples
•	192.168.1.10
•	10.0.0.1
•	172.16.5.20
•	8.8.8.8
________________________________________
Public IP Address
Definition
A Public IP Address is an IP address that is accessible over the Internet.
It is assigned by an Internet Service Provider (ISP).
Characteristics
•	Accessible from the Internet
•	Globally unique
•	Used for Internet communication
Example
Your Laptop
      │

      │

•	8.8.8.8
•	1.1.1.1
________________________________________
Private IP Address
Definition
A Private IP Address is used within a local network (LAN). It cannot be accessed directly from the Internet.
Private IPs are assigned by a router.
Characteristics
•	Used inside homes and offices
•	Not directly accessible from the Internet
•	Can be reused in different networks

Loopback Address
Definition
The Loopback Address is 127.0.0.1.
It is used to test the local computer's network interface.
Traffic sent to the loopback address never leaves the computer.
________________________________________
Localhost
Definition
Localhost is the hostname that refers to 127.0.0.1.
Developers and administrators use localhost to test applications running on the same computer.
Example
http://localhost
or
http://127.0.0.1
Both refer to the same local machine.

DNS (Domain Name System)
Definition
DNS (Domain Name System) is a system that translates domain names into IP addresses, allowing users to access websites using easy-to-remember names instead of numerical IP addresses.
Example
google.com
      ↓
142.250.xxx.xxx
________________________________________
Why DNS Exists
Humans remember domain names, while computers communicate using IP addresses.
DNS acts as a translator between humans and computers.
Without DNS, users would have to remember IP addresses for every website.
________________________________________
Why DNS is Important
•	Converts domain names into IP addresses.
•	Makes Internet browsing easy.
•	Allows users to access websites using names instead of numbers.
•	Reduces the need to memorize IP addresses.
________________________________________

Simple DNS Resolution Steps
1.	User enters google.com in the browser.
2.	Browser sends a DNS query.
3.	DNS Resolver searches for the IP address.
4.	DNS Server returns the IP address.
5.	Browser connects to the website using the IP address.
6.	The website loads.

DHCP (Dynamic Host Configuration Protocol)
Definition
DHCP (Dynamic Host Configuration Protocol) is a network protocol that automatically assigns IP addresses and other network settings to devices when they join a network.
It eliminates the need to manually configure network settings for every device.
________________________________________
Why is DHCP Needed?
Without DHCP:
•	Every device must be configured manually.
•	IP address conflicts can occur.
•	Network management becomes difficult.
•	More time is required to configure devices.
With DHCP:
•	IP addresses are assigned automatically.
•	Configuration is faster and easier.
•	Reduces human errors.
•	Prevents duplicate IP addresses.
________________________________________
What Does DHCP Assign?
A DHCP Server automatically provides:
•	IP Address
•	Subnet Mask
•	Default Gateway
•	DNS Server
These settings allow a device to communicate on the network.
________________________________________
How Devices Receive an IP Address
When a new device connects to a network:
1.	It searches for a DHCP Server.
2.	The DHCP Server offers an available IP address.
3.	The device accepts the offered IP.
4.	The DHCP Server confirms the assignment.
The device is now ready to communicate.
________________________________________
DHCP Process (DORA)
The DHCP process follows four steps known as DORA.
D – Discover
The device broadcasts a request to find a DHCP Server.
Purpose:
Search for an available DHCP Server.
________________________________________
O – Offer
The DHCP Server offers an available IP address.
Purpose:
Provide an IP address to the device.
________________________________________
R – Request
The device requests the offered IP address.
Purpose:
Accept the IP address offered by the DHCP Server.
________________________________________
A – Acknowledge
The DHCP Server confirms the IP address assignment.
Purpose:
Complete the IP assignment process.
